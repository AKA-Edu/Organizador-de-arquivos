# Product Requirement Document (PRD) - CLI Organizador de Arquivos

## 1. Visão Geral do Projeto
O objetivo deste projeto é construir uma ferramenta de linha de comando (CLI) em **TypeScript Puro** rodando no **Node.js**. A aplicação deve escanear uma pasta específica informada pelo usuário (como a pasta `Downloads`) e organizar automaticamente todos os arquivos contidos nela, movendo-os para subpastas categorizadas com base nas suas extensões (ex: `.pdf` vai para a pasta `Documentos`).

---

## 2. Escopo do Sistema

### O que o sistema DEVE fazer:
*   Receber o caminho de uma pasta do computador como argumento no terminal.
*   Validar se o caminho informado realmente existe e se é uma pasta válida.
*   Ler todos os arquivos presentes na raiz dessa pasta (ignorando subpastas já existentes).
*   Identificar a extensão de cada arquivo (ex: `.jpg`, `.mp3`, `.zip`).
*   Criar subpastas específicas (Imagens, Documentos, Compactados, etc.) apenas se houver arquivos para elas.
*   Mover os arquivos com segurança para suas respectivas subpastas.
*   Exibir um relatório final no terminal mostrando quantos arquivos foram organizados.

### O que o sistema NÃO DEVE fazer:
*   Não deve deletar nenhum arquivo sob nenhuma circunstância.
*   Não deve organizar subpastas de forma recursiva (ele mexe apenas nos arquivos soltos da pasta principal).
*   Não precisa de interface visual; a interação é feita 100% por comandos de texto no terminal.

---

## 3. Regras de Negócio (Mapeamento de Categorias)
O sistema deve seguir uma constante rígida de mapeamento para saber para onde enviar cada arquivo:

*   **Documentos**: `.pdf`, `.docx`, `.txt`, `.xlsx`, `.pptx`
*   **Imagens**: `.jpg`, `.jpeg`, `.png`, `.gif`, `.svg`
*   **Vídeos**: `.mp4`, `.mkv`, `.avi`, `.mov`
*   **Músicas**: `.mp3`, `.wav`, `.flac`
*   **Compactados**: `.zip`, `.rar`, `.7z`, `.tar`
*   **Outros**: Qualquer extensão que não se encaixe nas regras acima deve ir para uma pasta chamada `Outros`.

*Regra de Conflito:* Se um arquivo com o mesmo nome já existir na pasta de destino, o sistema deve renomear o novo arquivo adicionando um sufixo (ex: `foto_1.jpg`) para evitar que o arquivo antigo seja sobrescrito.

---

## 4. Tipagem Estrita (Contratos do TypeScript)
Para blindar o código contra erros de arquivos inexistentes ou caminhos nulos, definiremos:
*   `Categoria`: Um tipo literal restrito a `'Documentos' | 'Imagens' | 'Videos' | 'Musicas' | 'Compactados' | 'Outros'`.
*   `MapeamentoExtensoes`: Um tipo de objeto onde a chave é a extensão (string) e o valor é a `Categoria` correspondente (`Record<string, Categoria>`).
*   `ResultadoOrganizacao`: Interface para o relatório final `{ totalMovidos: number; erros: number }`.

---

## 5. Fluxo de Execução Lógica (Algoritmo)

Este fluxo desenha a lógica necessária para lidar com o sistema de arquivos de forma assíncrona (`async/await`) usando o módulo nativo `node:fs/promises`:

```text
[Início]
   │
   ▼
[Capturar o caminho da pasta via argumento do terminal: process.argv]
   │
   ▼
[A pasta existe?] ───(NÃO)───► [Exibir erro: "Pasta não encontrada"] ──► [Fim]
   │ (SIM)
   ▼
[Ler todos os itens de dentro da pasta (fs.readdir)]
   │
   ▼
[Filtrar a lista: Manter apenas ARQUIVOS, ignorar PASTAS]
   │
┌──┴────────────────────────────────┐
│ LOOP PARA CADA ARQUIVO ENCONTRADO │ ◄──────────────────────────────┐
└──┬────────────────────────────────┘                                │
   ▼                                                                 │
[Extrair a extensão do arquivo (ex: .png) usando node:path]          │
   │                                                                 │
   ▼                                                                 │
[Descobrir a Categoria correspondente no dicionário de tipos]        │
   │                                                                 │
   ▼                                                                 │
[A subpasta da Categoria já existe? (Ex: /Imagens)]                  │
   ├───► (NÃO) ──► [Criar a subpasta usando fs.mkdir]                │
   └───► (SIM) ──► [Seguir em frente]                                │
   │                                                                 │
   ▼                                                                 │
[O nome do arquivo já existe na pasta de destino?]                   │
   ├───► (SIM) ──► [Modificar o nome gerando um novo Path]           │
   └───► (NÃO) ──► [Manter Path original]                            │
   │                                                                 │
   ▼                                                                 │
[Mover o arquivo para o novo destino (fs.rename)]                    │
   │                                                                 │
   ▼                                                                 │
[Incrementar contador de Sucesso (ou de Erro se falhar a permissão)] │
   │                                                                 │
   ▼                                                                 │
[Existem mais arquivos na lista?] ───(SIM)───────────────────────────┘
   │ (NÃO)
   ▼
[Exibir Relatório final com o total de arquivos movidos com sucesso]
   │
   ▼
[Encerrar Processo]
```
